## 7. 잘한 점 (살릴 강점)

✅ **문제와 사용자가 한 쌍으로 잘 맞물림**
- 「출근 전 10분 안에 시장 전망을 파악하고 투자 결정을 하고 싶은 초심 투자자」vs 「매일 아침 체계적인 시장 분석을 통해 포트폴리오를 관리하고 싶은 경험 많은 투자자」
- 두 페르소나의 니즈가 "매일 아침 빠른 시장 브리핑"이라는 한 가치로 모임
- → 시장 정의가 명확하고 설득력 있음

✅ **AI 활용 지점이 분명함**
- 「Claude API를 활용해 전날 경제뉴스를 분석 → 오늘의 시장 전망 1문단 + 투자포인트 3-5개 자동 생성」
- 「각 뉴스를 Claude API로 1-2문단 요약」
- → AI가 무엇을 만드는지, 왜 필요한지가 명확

✅ **범위가 현실적 (1인·3주 기준)**
- 「웹사이트(전망·뉴스) + 이메일 구독」 두 축으로 좁음
- 거시지표·기업공시·기술적 분석을 「v2~v3」로 명확히 미룸
- → 스코핑이 탄탄함

✅ **데이터 흐름이 자동화 중심으로 단순함**
- 「네이버뉴스 API → Claude → APScheduler → 이메일」 파이프라인이 깔끔
- 표준 라이브러리(FastAPI, anthropic, APScheduler)로 1인 개발에 최적
- → 구현이 자명하고 변수가 적음

✅ **우선순위가 명확함**
- 뉴스 요약 + 시장 분석이 핵심 (이메일은 부가)
- 웹사이트를 블로그처럼 표시하는 구체적 이미지 있음
- → 개발 방향이 흔들리지 않을 것

✅ **환경 제약이 없음**
- API 호출, 라이브러리 설치 자유
- 기술적 막힘이 없음
- → Claude Code + 외부 도구로 빠르게 해결 가능

## 8. 부족한 점 (완주를 위협하는 것)

⚠️ **시간이 너무 부족** (1순위)
- 실제 가용: 20~25시간
- SPEC 기준: 56시간
- **격차: 약 40%**
- → Phase 1만 하거나, Phase 1·2를 쪼갠 뒤 일부를 제외해야 함

⚠️ **Python/FastAPI 미경험** (2순위)
- 웹앱 경험은 있지만 프레임워크가 새로움
- 문법, 라이브러리, 패턴 학습 필요
- Claude/GPT에 물어봐도 1~2시간이 소모될 수 있음
- **선택지:**
  - Python: 익숙한 AI 라이브러리 (anthropic-sdk) ✅
  - Node.js: 더 빠른 개발 속도 (기존 경험 활용) ⭐ 추천

⚠️ **콘텐츠 품질·신뢰성 검증이 빠져 있음** (3순위, 가장 중요)
- 기술 구현은 어렵지 않지만, **성공의 척도가 콘텐츠 품질**
- 성공 기준 3번: 「생성된 보고서 내용이 실제 투자 결정에 도움이 될 만큼의 인사이트 제공」
- 현재 SPEC에 品質 검증 절차가 없음
- 필수 검증 사항:
  - **할루시네이션 규율:** AI가 뉴스에 없는 수치·전망을 만들지 않는가?
  - **단정성 제어:** 예측이 근거 있고 조건부인가?
  - **출처 명시:** 어떤 뉴스에 근거했는지 명확한가?
  - **실제 테스트:** 최근 뉴스 5일치로 생성한 결과가 정말 투자자가 신뢰할 수 있는가?

⚠️ **금융 콘텐츠의 리스크 규율이 필요**
- 투자 전망은 틀리거나 단정적이면 신뢰를 한순간에 잃음
- 작성 원칙 필요:
  - 「전날 뉴스에 없는 수치·전망을 만들지 않는다」
  - 「단정적 예측 대신 근거와 함께 제시한다」
  - 「투자 참고용 면책 고지」 필수
- → 이것들이 있으면 신뢰성과 안정성이 함께 올라감

⚠️ **외부 의존성이 데모 변수**
- 네이버뉴스 API 응답이 불안정할 수 있음
- 매일 아침 6시 스케줄 발행은 시연 당일 돌아가지 않음
- HTML 이메일 포맷은 발신 도메인·스팸 필터 변수 있음
- → 데모에서 리스크 있음

⚠️ **Claude API 신뢰도/비용/보안 불확실**
- 뉴스 분석 결과가 항상 좋은 품질인가?
- 비용이 예상을 초과할 수 있나?
- 보안 위험은?
- → **7/15 이전에 반드시 테스트 필수**
