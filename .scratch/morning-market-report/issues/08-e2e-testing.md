## 구축할 내용

웹사이트 전체가 정상 작동하는지 확인하고, PRD에서 강조한 신뢰성 규율이 지켜지고 있는지 검증합니다. 할루시네이션, 편향성, 규제 언어 준수 여부를 수동으로 점검하고 데모 시나리오를 준비합니다.

## 수락 기준

- [ ] "지금 생성" 버튼 클릭 → 요약 생성 → 웹사이트 표시 (E2E)
- [ ] 생성된 요약 5개가 원뉴스와 대조했을 때 할루시네이션 없음 확인
- [ ] 투자 권유 표현 ("사세요", "반드시") 없음 확인
- [ ] 조건부 표현만 사용됨 ("~할 가능성", "주목할 만하다") 확인
- [ ] 편향성 검증: 금융/IT/바이오/에너지 등 섹터별로 고르게 다루는지 확인
- [ ] 모든 뉴스에 출처(제목, 언론사, 발행일, 링크) 명시됨 확인
- [ ] 면책 고지가 명확하게 표시됨 확인
- [ ] Claude API 비용 (일일/월별) 측정 가능하도록 로깅 확인
- [ ] 7/15 시연을 위한 데모 스크립트 준비
  - 최근 뉴스 5개 선정
  - "지금 생성" 버튼 클릭 시연
  - 1~2개 요약을 읽어주고 원문과 비교
  - 신뢰성 설득 포인트 정리

## 기술 노트

**신뢰성 검증 체크리스트:**

**할루시네이션 검증:**
- 요약의 모든 수치·통계가 원문에 존재하는가?
- 요약에만 있는 예측이나 주장은 없는가?

**단정성 검증:**
- "이 주식을 사세요", "반드시 상승한다" 같은 표현이 없는가?
- 모든 표현이 "~할 가능성", "~는 주목할 만하다" 등 조건부인가?

**편향성 검증:**
- 금융 관련 뉴스 몇 개?
- IT 관련 뉴스 몇 개?
- 바이오/에너지 등 다른 섹터는?
- 특정 업종이 과도하게 강조되지 않았는가?

**출처 명시 검증:**
- 모든 뉴스에 제목, 언론사, 발행일, 링크가 있는가?
- 링크를 클릭하면 원문으로 이동하는가?

**규제 안전성:**
- 면책 고지가 명확한가?
- 서비스가 "투자 참고용"임이 명백한가?

**성능:**
- "지금 생성" 클릭부터 결과 표시까지 5분 이내인가?
