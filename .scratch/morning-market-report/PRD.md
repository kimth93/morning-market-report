# PRD: 아침 시장 전망 보고서 자동화 플랫폼

**상태:** Ready for Implementation  
**버전:** 2.0  
**목표 기한:** 2026-07-15 (3주)  
**최소 노력:** 약 20시간

---

## Problem Statement

금융권 종사자와 일반 투자자는 매일 아침 시장 동향을 파악하는 데 상당한 시간을 투자합니다. 특히:

- **금융권 종사자**: 매일 보고서 작성에 소요되는 시간 단축 필요
- **일반 투자자**: 복잡한 경제뉴스를 빠르게 파악하고 당일 투자전략을 수립하기 어려움

현재 이들은 여러 뉴스매체를 수동으로 방문하여 뉴스를 읽고 요약하는 반복적인 작업을 수행하고 있습니다.

---

## Solution

**아침 시장 전망 보고서 자동화 플랫폼**은 Claude AI를 활용하여 경제/증권 뉴스를 자동으로 수집 및 요약하고, 신뢰도 높은 인사이트와 투자포인트를 제공하는 웹 기반 서비스입니다.

**핵심 가치:**
1. **신속성**: 매일 아침 5분 내에 시장 요약 획득
2. **신뢰성**: 출처 명시 + 할루시네이션 제어 + 편향성 관리
3. **실용성**: 투자 결정에 필요한 핵심 정보 제공

---

## User Stories

1. As a 20대 직장인 (초심 투자자), I want to 출근 전 10분 안에 시장 전망을 파악하고 투자 결정을 하기, so that 바쁜 아침에도 투자 기회를 놓치지 않을 수 있다

2. As a 30대 자산가 (경험 많은 투자자), I want to 매일 아침 체계적인 시장 분석을 통해 포트폴리오를 관리하기, so that 데이터 기반의 투자 결정을 할 수 있다

3. As a 사용자, I want to 뉴스 Top 5를 1-2문단으로 요약한 내용을 보기, so that 원문을 읽을 시간이 없어도 핵심을 파악할 수 있다

4. As a 사용자, I want to 요약된 뉴스의 출처(제목, 언론사, 발행일, 링크)를 명확하게 확인하기, so that 필요하면 즉시 원문으로 이동해 더 자세히 읽을 수 있다

5. As a 신중한 투자자, I want to 생성된 요약이 원뉴스에 없는 수치나 통계를 만들어내지 않는지 확인하기, so that AI 할루시네이션의 위험을 최소화할 수 있다

6. As a 사용자, I want to 웹사이트가 "투자는 본인의 판단으로" 같은 명확한 면책 고지를 표시하기, so that 이 서비스가 투자 참고용일 뿐이라는 점을 이해할 수 있다

7. As a 사용자, I want to 시장 전망을 읽기, so that 오늘의 시장 방향성을 예측할 수 있다

8. As a 사용자, I want to 투자포인트 3개를 보기, so that 특별히 주목해야 할 종목이나 섹터를 파악할 수 있다

9. As a 이메일 구독에 관심 있는 사용자, I want to 구독 신청 form을 작성하기, so that 앞으로 매일 아침 자동으로 보고서를 받을 수 있다 (v2에서 실제 발송)

10. As a 사용자, I want to 구독했을 때 받을 이메일의 예시 화면을 미리 보기, so that 서비스의 최종 형태를 이해할 수 있다

11. As a 개발자, I want to 웹사이트 UI/디자인이 명확하고 읽기 편하기, so that 사용자가 정보에 쉽게 접근할 수 있다

12. As a 개발자, I want to "지금 생성" 버튼을 클릭하면 실시간으로 요약이 생성되기, so that 사용자가 최신 뉴스를 즉시 확인할 수 있다

13. As a 플랫폼 관리자, I want to 요약에 투자 권유 표현(예: "이 주식을 사세요")이 없는지 검증하기, so that 규제 위험을 회피할 수 있다

14. As a 플랫폼 관리자, I want to 섹터별(금융, IT, 바이오, 에너지 등) 뉴스 다루기에 편향이 없는지 확인하기, so that 신뢰도 높은 플랫폼이라는 평판을 유지할 수 있다

15. As a 성능 중시 사용자, I want to 뉴스 요약 생성 시간이 충분히 빠르기, so that 기다리는 시간 없이 결과를 확인할 수 있다

## Implementation Decisions

### 1. 기술 스택

**백엔드: Python + FastAPI**
- Claude API 공식 SDK (anthropic-sdk) 활용
- LLM 기반 분석에 최적화된 라이브러리 생태계
- 스케줄링, 데이터 처리 표준 라이브러리 우수

**주요 라이브러리:**
- FastAPI (웹 API 서버)
- anthropic (Claude API)
- requests (네이버뉴스 API)
- SQLAlchemy (SQLite ORM)
- python-dotenv (환경 변수)

**프론트엔드: React**
- 뉴스 요약 + 시장 전망 표시
- UI/디자인은 개발 중 결정
- TypeScript 선택사항

**데이터베이스: SQLite**
- 로컬 개발 환경에 최적
- 향후 PostgreSQL로 마이그레이션 가능

### 2. Phase 1: 뉴스 요약 (MVP 핵심)

**뉴스 Top 5 요약 기능:**
- 네이버뉴스 API에서 경제/증권 뉴스 수집
- Claude API로 각 뉴스를 1-2문단으로 요약
- **필수 정보:** 뉴스 제목 + 언론사 + 발행일 + 원문 링크 명시
- 웹 UI에 표시

**당일 시장 전망 & 투자포인트:**
- 전날 뉴스 기반 시장 전망 1문단 생성 (Claude API)
- 투자포인트 3개 (프로토타입 형태로도 OK)
- 신뢰성 규율 준수

**웹 인터페이스:**
- 필수: 뉴스 요약 5개 표시
- 선택: 반응형 디자인 (시간 있으면)
- 필수: 출처 + 면책 고지 명시 ("투자는 본인의 판단으로")

### 3. Phase 2: 이메일 구독 (예시 형태만, 실제 발송은 v2)

**이메일 구독 신청:**
- 웹사이트에 구독 신청 form (시각적)
- 실제 메일 발송은 v2에서 구현

**예시 이메일 화면:**
- 구독했을 때 받을 이메일의 예시 화면 표시
- 실제 자동 발송은 7/15 시연 이후

### 4. 신뢰성 규율 (반드시 준수)

#### 4-1. 할루시네이션 금지
- **정의:** AI가 원문에 없는 수치·통계·예측을 만드는 것
- **원칙:** 모든 수치와 주장은 원뉴스에 근거해야 함
- **검증:** 생성된 요약의 모든 수치를 원뉴스와 대조

#### 4-2. 단정성 제어
- **금지:** "이 주식을 사세요", "반드시 상승한다" 같은 직접적 투자 권유
- **허용:** "~할 가능성이 있다", "~는 주목할 만하다", "~를 지켜볼 필요가 있다" (조건부 표현)
- **면책 고지:** 웹사이트 + 요약 상단에 **"투자는 본인의 판단으로 하시기 바랍니다"** 명시

#### 4-3. 편향성 제어
- **정의:** 특정 업종/종목만 과도하게 강조하거나 부정적 표현
- **원칙:** 원뉴스의 균형과 톤을 유지
- **검증:** 금융/IT/바이오/에너지 등 섹터별로 고르게 다루는지 확인

#### 4-4. 출처 명시 (필수)
- **포함 내용:** 뉴스 제목 + 언론사 + 발행일 + 원문 링크
- **위치:** 요약 아래 또는 옆에 명확하게 표시
- **목적:** 사용자가 한 번에 원문 확인 가능

### 5. 자동화 범위 (v1에서는 수동)

- ❌ APScheduler (자동 스케줄) → **"지금 생성" 버튼으로 대체** (v2에서 APScheduler 구현)
- ❌ 이메일 실제 자동 발송 (v2에서)
- ❌ 복잡한 이메일 HTML 템플릿 (v2에서)

### 6. API 계약

**뉴스 생성 엔드포인트:**
```
POST /api/news/generate
Response: {
  "news_items": [
    {
      "title": "string",
      "source": "string",
      "date": "string",
      "url": "string",
      "summary": "string",
      "hallucination_risk": "low|medium|high"
    }
  ],
  "market_outlook": "string",
  "investment_points": ["string", "string", "string"],
  "generated_at": "timestamp"
}
```

**이메일 구독 엔드포인트:**
```
POST /api/subscription/signup
Body: {
  "email": "string",
  "name": "string"
}
Response: {
  "status": "success",
  "subscription_id": "string"
}
```

## Testing Decisions

### 좋은 테스트의 정의
- **외부 동작만 테스트**: 내부 구현이 아닌 사용자가 관찰할 수 있는 결과를 검증
- **신뢰성 규율 검증**: 할루시네이션, 단정성, 편향성, 출처 명시 여부 확인
- **실제 뉴스 기반**: 실제 경제뉴스를 사용한 통합 테스트

### 테스트 대상 모듈

1. **뉴스 수집 모듈**
   - 네이버뉴스 API에서 정상적으로 뉴스 수집 여부
   - 수집한 뉴스의 필수 필드(제목, 언론사, 링크) 존재 여부

2. **Claude API 요약 모듈**
   - 요약의 길이가 1-2문단인지 확인
   - 할루시네이션 여부 (원뉴스와의 수치 대조)
   - 직접적 투자 권유 표현 제거 여부
   - 조건부 표현 사용 여부

3. **출처 명시 모듈**
   - 모든 요약에 제목, 언론사, 발행일, 링크가 함께 표시되는지 확인

4. **시장 전망 & 투자포인트 모듈**
   - 전망 문장이 생성되는지 확인
   - 투자포인트 3개가 모두 생성되는지 확인

5. **웹 UI 통합 테스트**
   - "지금 생성" 버튼 클릭 시 요약이 렌더링되는지 확인
   - 면책 고지가 보이는지 확인
   - 모든 뉴스 요약이 화면에 표시되는지 확인

### 테스트 전략

- **수동 검증** (7/15 전): 실제 뉴스 5개를 요약하고 원뉴스와 대조
- **단위 테스트**: Claude API 응답 형식 및 필드 검증
- **통합 테스트**: 엔드투엔드 파이프라인 (수집 → 요약 → 표시)
- **신뢰성 검증 체크리스트**: 할루시네이션, 단정성, 편향성, 출처 명시

## Out of Scope

### v1에서 제외 사항

- ❌ **APScheduler**: 자동 스케줄링 (대신 "지금 생성" 수동 버튼 사용)
- ❌ **이메일 자동 발송**: 실제 메일 발송 기능 (v2에서)
- ❌ **복잡한 이메일 HTML 템플릿**: 기본 형태만 예시 (v2에서)
- ❌ **거시경제 심층 분석**: 한국은행 EOOS, FRED API (v2에서)
- ❌ **기업 공시 데이터**: DART, KRX API (v2에서)
- ❌ **복잡한 기술적 분석**: 차트, 이동평균선 등 (v3에서)
- ❌ **개인화 포트폴리오 추적**: 로그인, 사용자 데이터 저장 (v3에서)
- ❌ **실시간 주가 데이터**: 주가 연동 (v2에서)
- ❌ **복잡한 UI/디자인**: 기본 가독성만 확보 (추후 리디자인)

## Success Criteria

1. ✅ 웹사이트에 뉴스 Top 5 요약이 표시됨
2. ✅ 생성된 요약이 원뉴스와 대조했을 때 할루시네이션이 없다
3. ✅ 투자 권유 표현이 없고 조건부 표현으로만 되어 있다
4. ✅ 섹터별 편향이 없다
5. ✅ 모든 요약에 출처가 명확하게 표시된다
6. ✅ 면책 고지가 명확하게 표시된다
7. ✅ 시장 전망 + 투자포인트가 프로토타입 형태로 나온다 (완성도는 낮아도 OK)
8. ✅ 이메일 구독 form + 예시 화면이 있다
9. ✅ "지금 생성" 버튼 클릭 시 실시간으로 요약이 생성되고 표시된다
10. ✅ Claude API 비용이 일일/월별 기준으로 측정 가능하다
