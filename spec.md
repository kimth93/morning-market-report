# MVP Spec: 아침 시장 전망 보고서 자동화 플랫폼

## 1. 문제
- **금융권 종사자**: 매일 보고서 작성에 소요되는 시간 단축 필요
- **일반 투자자**: 복잡한 경제뉴스를 빠르게 파악하고 당일 투자전략을 수립하기 어려움

## 2. 사용자 (페르소나)
- **20대 직장인**: 출근 전 10분 안에 시장 전망을 파악하고 투자 결정을 하고 싶은 초심 투자자
- **30대 자산가**: 매일 아침 체계적인 시장 분석을 통해 포트폴리오를 관리하고 싶은 경험 많은 투자자

## 3. 핵심 기능 (MVP)

### Phase 1: 웹사이트 (필수)
1. **당일 시장 전망 & 투자포인트**
   - Claude API를 활용해 전날 경제뉴스를 분석
   - 오늘의 시장 전망 1문단 + 투자포인트 3-5개 자동 생성
   - 웹사이트에 매일 아침 6시 업데이트

2. **주요 경제뉴스 요약 (Top 5)**
   - 네이버뉴스 API로 경제/증권 카테고리 뉴스 수집
   - 각 뉴스를 Claude API로 1-2문단 요약
   - 제목, 요약, 원문 링크 표시

3. **웹 인터페이스**
   - React로 간단한 대시보드 구현
   - 당일 시장 전망, 투자포인트, Top 5 뉴스 요약 표시
   - 반응형 디자인 (모바일 최적화)

### Phase 2: 이메일 구독 (필수)
4. **일일 보고서 이메일 발송**
   - 사용자가 웹에서 이메일 구독 신청 가능
   - 매일 아침 6시 30분에 자동 발송
   - 포함 내용: 시장 전망 + 투자포인트 + 뉴스 요약
   - 깔끔한 HTML 이메일 포맷

## 4. 제외 범위 (추후 확장)
- ❌ 거시경제 심층 분석 (한국은행 EOOS, FRED API 통합은 v2에서)
- ❌ 기업 공시 데이터 분석 (DART, KRX API는 v2에서)
- ❌ 복잡한 기술적 분석 (차트, 이동평균선 등 기술지표)
- ❌ 개인화 포트폴리오 추적 (로그인, 사용자 데이터 저장)
- ❌ 실시간 주가 데이터 연동 (스크린샷으로 충분)

## 5. 성공 기준
1. ✅ 웹사이트에 매일 오전 6시까지 당일 시장 전망 + 투자포인트 업데이트됨
2. ✅ 경제뉴스 Top 5가 요약되어 사용자가 5분 안에 읽을 수 있음
3. ✅ 생성된 보고서 내용이 실제 투자 결정에 도움이 될 만큼의 인사이트 제공
4. ✅ 이메일 구독 기능이 정상 작동하고 매일 발송됨
5. ✅ 모바일에서도 웹사이트가 정상 표시됨

## 6. 기술 스택

### 백엔드: **Python** (추천)
**선택 이유:**
- Claude API 통합 및 LLM 기반 분석이 간편 (anthropic-sdk)
- 뉴스 API, 이메일 자동화, 데이터 처리가 표준 라이브러리로 가능
- 스케줄 작업 (APScheduler)이 강력함
- 1주일 1명 개발에 최적

**주요 라이브러리:**
- FastAPI (웹 API 서버)
- anthropic (Claude API)
- requests (네이버뉴스 API)
- APScheduler (일일 스케줄)
- SQLAlchemy (PostgreSQL ORM)
- python-dotenv (환경 변수)

### 프론트엔드: **React**
- Next.js 추천 (SSR, 배포 간편)
- TypeScript (선택사항이지만 추천)
- Tailwind CSS 또는 Material-UI (빠른 UI 구축)

### 데이터베이스: **PostgreSQL**
- 사용자 이메일, 구독 여부, 뉴스 히스토리 저장
- 테이블: users, subscriptions, daily_reports, news_summaries

### 배포
- 백엔드: Railway, Render, 또는 AWS Lambda
- 프론트: Vercel (Next.js 최적)
- 데이터베이스: Railway PostgreSQL 또는 AWS RDS

## 7. 개발 일정 (1주일)

| 일차 | 작업 | 산출물 |
|------|------|--------|
| Day 1-2 | 백엔드 구조 설계 + PostgreSQL 테이블 설계 + Claude API 연동 | API 기본 구조 |
| Day 2-3 | 네이버뉴스 API 연동 + 뉴스 요약 기능 구현 | 뉴스 수집 & 요약 API |
| Day 3-4 | 당일 시장 전망 생성 로직 + APScheduler 설정 | 일일 업데이트 자동화 |
| Day 4-5 | 이메일 자동 발송 기능 + 사용자 구독 API | 이메일 시스템 |
| Day 5-6 | React 프론트엔드 UI 구축 | 웹사이트 |
| Day 6-7 | 통합 테스트 + 배포 + 버그 수정 | 프로덕션 배포 |

## 8. 향후 확장 (v2, v3)
- v2: 거시경제 지표 연동 (한국은행 EOOS, FRED API)
- v3: 기업 공시 분석 (DART, KRX API)
- v4: 개인화 포트폴리오 추적 및 AI 기반 맞춤 분석
- v5: 모바일 앱 (React Native)
