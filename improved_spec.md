# 개선된 MVP Spec: 아침 시장 전망 보고서 자동화 플랫폼

**버전:** 2.0 (개선 모드 완료)
**개선 일시:** 2026-06-24
**목표:** 3주(7/15) 내 신뢰도 있는 뉴스 요약 플랫폼 완성

---

## 1. 문제

- **금융권 종사자**: 매일 보고서 작성에 소요되는 시간 단축 필요
- **일반 투자자**: 복잡한 경제뉴스를 빠르게 파악하고 당일 투자전략을 수립하기 어려움

---

## 2. 사용자 (페르소나)

- **20대 직장인**: 출근 전 10분 안에 시장 전망을 파악하고 투자 결정을 하고 싶은 초심 투자자
- **30대 자산가**: 매일 아침 체계적인 시장 분석을 통해 포트폴리오를 관리하고 싶은 경험 많은 투자자

---

## 3. 핵심 기능 (MVP)

### ✅ Phase 1: 웹사이트 (필수)

#### 1-1. 뉴스 Top 5 요약 (완성도 높게 ⭐ 최우선)

- Claude API를 활용해 경제/증권 뉴스 분석
- 각 뉴스를 1-2문단으로 자동 요약
- **출처 명시**: 뉴스 제목 + 언론사 + 발행일 + 원문 링크 필수 표시
- 신뢰성 규율 준수 (아래 참고)
- 웹사이트에 표시
- **UI/디자인:** 개발 단계에서 결정 (블로그 형식 또는 대시보드)

#### 1-2. 당일 시장 전망 & 투자포인트 (프로토타입 형태 ⏳)

- Claude API로 전날 뉴스 기반 시장 전망 1문단 생성
- 투자포인트 3개 (프로토타입 형태로도 OK)
- 신뢰성 규율 준수

#### 1-3. 웹 인터페이스 (기본 기능만)

- **필수:** 뉴스 요약 5개 표시
- **선택:** 반응형 디자인 (시간 있으면)
- **포함:** 출처 + 면책 고지 명시 ("투자는 본인의 판단으로")

---

### 📋 Phase 2: 이메일 구독 (예시 형태만)

#### 2-1. 이메일 구독 신청 (UI만)

- 웹사이트에 구독 신청 form (시각적)
- 실제 메일 발송은 **v2에서** (우선순위 낮음)

#### 2-2. 예시 이메일 화면

- 구독했을 때 받을 이메일의 **예시 화면** 표시
- 실제 자동 발송은 7/15 시연 이후

---

## 4. 신뢰성 규율 (필수)

뉴스 요약 생성 시 다음 4가지를 **반드시 지킨다**:

### 4-1. 할루시네이션 금지 ❌

- **정의:** AI가 원문에 없는 수치·통계·예측을 만드는 것
- **원칙:** 모든 수치와 주장은 원뉴스에 근거해야 함
- **검증 방법:** 생성된 요약의 모든 수치를 원뉴스와 대조

### 4-2. 단정성 제어 ❌

- **금지:** "이 주식을 사세요", "반드시 상승한다" 같은 직접적 투자 권유
- **허용:** "~할 가능성이 있다", "~는 주목할 만하다", "~를 지켜볼 필요가 있다" (조건부 표현)
- **면책 고지:** 웹사이트 + 요약 상단에 **"투자는 본인의 판단으로 하시기 바랍니다"** 명시

### 4-3. 편향성 제어 ❌

- **정의:** 특정 업종/종목만 과도하게 강조하거나 부정적 표현
- **원칙:** 원뉴스의 균형과 톤을 유지
- **검증 방법:** 금융/IT/바이오/에너지 등 섹터별로 고르게 다루는지 확인

### 4-4. 출처 명시 (필수) ✅

- **포함 내용:** 뉴스 제목 + 언론사 + 발행일 + 원문 링크
- **위치:** 요약 아래 또는 옆에 명확하게 표시
- **목적:** 사용자가 한 번에 원문 확인 가능

---

## 5. 성공 기준

1. ✅ 웹사이트에 뉴스 Top 5 요약이 표시됨
2. ✅ 생성된 요약이 원뉴스와 대조했을 때 할루시네이션이 없다
3. ✅ 투자 권유 표현이 없고 조건부 표현으로만 되어 있다
4. ✅ 섹터별 편향이 없다
5. ✅ 모든 요약에 출처가 명확하게 표시된다
6. ✅ 면책 고지가 명확하게 표시된다
7. ✅ 시장 전망 + 투자포인트가 프로토타입 형태로 나온다 (완성도는 낮아도 OK)
8. ✅ 이메일 구독 form + 예시 화면이 있다

---

## 6. 제외 범위 (추후 확장)

- ❌ APScheduler (자동 스케줄) → 수동 "지금 생성" 버튼으로 대체 (v2에서 구현)
- ❌ 이메일 실제 자동 발송 (v2에서)
- ❌ 복잡한 이메일 HTML 템플릿 (v2에서)
- ❌ 거시경제 심층 분석 (한국은행 EOOS, FRED API는 v2)
- ❌ 기업 공시 데이터 분석 (DART, KRX API는 v2)
- ❌ 복잡한 기술적 분석 (차트, 이동평균선 등)
- ❌ 개인화 포트폴리오 추적 (로그인, 사용자 데이터 저장)
- ❌ 실시간 주가 데이터 연동

## 7. 기술 스택

### 백엔드: **Python + FastAPI** ⭐

**선택 이유:**

- Claude API 공식 SDK (anthropic-sdk) 활용
- LLM 기반 분석에 최적화된 라이브러리
- 스케줄링, 데이터 처리 표준 라이브러리 우수
- AI 생태계가 가장 풍부

**주요 라이브러리:**

- FastAPI (웹 API 서버)
- anthropic (Claude API)
- requests (네이버뉴스 API)
- SQLAlchemy (SQLite ORM)
- python-dotenv (환경 변수)

### 프론트엔드: **React**

- 뉴스 요약 + 시장 전망 표시
- UI/디자인은 개발 중 결정
- TypeScript 선택사항

### 데이터베이스: **SQLite** (로컬용)

- 가볍고 로컬 개발에 최적
- PostgreSQL 대신 단순화

### 배포

- **로컬 작동** (필수)
- 실제 배포는 시간 있으면 (선택사항)

## 9. 7/15 시연 계획

### 시연 흐름

1. **"지금 생성" 버튼 클릭**
   - 네이버뉴스에서 경제뉴스 5개 수집
   - Claude API로 요약 생성 (실시간)
   - 웹사이트에 즉시 표시

2. **콘텐츠 품질 강조**
   - 생성된 요약 1~2개를 실제로 읽어주기
   - "이 요약이 원뉴스와 일치한다" 강조
   - 출처 링크 클릭해서 원문 확인

3. **신뢰성 설득**
   - "투자 참고용" 면책 고지 보여주기
   - "할루시네이션을 규율하는 작성 원칙" 설명
   - "출처 기반의 근거 있는 분석"임을 입증

4. **시장 전망 (프로토타입)**
   - 간단한 형태의 시장 전망 + 투자포인트 3개 보여주기

5. **이메일 (예시)**
   - 이메일 구독 form 보여주기
   - "이렇게 매일 받을 이메일" 예시 화면 표시

## 10. 7/15 이전 필수 확인 사항

- [ ] **Claude API 테스트** (비용/품질/신뢰도)
  - 뉴스 5개를 실제로 요약하고 출처와 비교
  - 비용 계산 (일일/월별)
  - 응답 시간 측정
  - 할루시네이션 체크

- [ ] **기술 검증**
  - FastAPI 기본 튜토리얼 1시간 (학습)
  - React 기억 되살리기 (필요하면)
  - 네이버뉴스 API 정상 작동 확인

- [ ] **신뢰성 규율 점검**
  - 생성된 요약이 할루시네이션 없는가?
  - 단정적 표현이 없는가?
  - 편향이 없는가?
  - 출처가 명확한가?

- [ ] **데모 준비**
  - 수동 "지금 생성" 버튼 작동 확인
  - 최근 뉴스 1~2일치로 시연 스크립트 준비
